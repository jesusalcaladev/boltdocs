# `getTranslated`

The `getTranslated` utility is used to resolve a value that can be either a simple string or an object containing multiple translations (locales). This is essential when building internationalized (i18n) layouts.

## Usage

You can import `getTranslated` from `boltdocs/client` (for layouts) or `boltdocs` (for node-side scripts).

```tsx
import { getTranslated } from 'boltdocs/client'

const title = getTranslated(config.theme?.title, 'en')
```

### Parameters

<Field name="value" type="string | Record<string, string> | undefined">
  The value to translate. Can be a string, an object like `{ en: 'Hello', es: 'Hola' }`, or undefined.
</Field>

<Field name="locale" type="string">
  The active locale key (e.g., `'en'`, `'es'`).
</Field>

---

## How it works

The function follows a specific resolution order to ensure a string is always returned:

1. **Direct String**: If `value` is already a `string`, it returns it as is.
2. **Locale Match**: If `value` is an object and contains a key matching the provided `locale`, it returns that translation.
3. **Fallback**: If no match is found, it returns the **first available** translation in the object.
4. **Empty**: If `value` is null or undefined, it returns an empty string `''`.

---

## Real World Example

In a custom `layout.tsx`, you can use it along with `useRoutes` to handle the site title dynamically:

```tsx
import { useRoutes, getTranslated, useConfig } from 'boltdocs/client'

export default function Layout({ children }) {
  const { currentLocale } = useRoutes()
  const config = useConfig()

  const siteTitle = getTranslated(config.theme?.title, currentLocale)

  return (
    <div>
      <h1>{siteTitle}</h1>
      {children}
    </div>
  )
}
```
