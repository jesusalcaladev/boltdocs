# useRoutes

`useRoutes` is the fundamental hook in Boltdocs. It provides access to every page in your project, automatically filtered by the user's current **Locale** and **Version**.

---

## Usage

Most other hooks and primitives depend on `useRoutes` to understand the site structure.

```tsx
import { useRoutes } from 'boltdocs/hooks'

function MyComponent() {
  const { routes, currentRoute } = useRoutes()
  
  return (
    <div>
      <h1>Current Page: {currentRoute?.title}</h1>
      <ul>
        {routes.map(route => (
          <li key={route.path}>{route.title}</li>
        ))}
      </ul>
    </div>
  )
}
```

---

## Return Value

| Property | Type | Description |
| :--- | :--- | :--- |
| `routes` | `Route[]` | The list of pages matching the active version and locale. |
| `allRoutes` | `Route[]` | Every single page in the project (unfiltered). |
| `currentRoute` | `Route \| null` | Metadata for the page currently being viewed. |
| `currentLocale` | `string` | The active language code (e.g., `"en"`). |
| `availableLocales` | `Locale[]` | List of languages configured in `boltdocs.config.ts`. |
| `currentVersion` | `string` | The active version path (e.g., `"v1"`). |
| `availableVersions` | `Version[]` | List of versions configured in `boltdocs.config.ts`. |

---

## Route Object API

A `Route` object contains everything Boltdocs knows about a Markdown file:

<Field name="title" type="string">
  The page title (from frontmatter or filename).
</Field>

<Field name="path" type="string">
  The URL path (e.g., `/docs/guides/getting-started`).
</Field>

<Field name="filePath" type="string">
  The relative path to the source file (e.g., `guides/getting-started.mdx`).
</Field>

<Field name="headings" type="TOCItem[]">
  An array of h2 and h3 headings extracted from the content.
</Field>

<Field name="frontmatter" type="Record<string, any>">
  The raw YAML frontmatter from the top of the file.
</Field>

<Field name="group" type="string">
  The group ID this page belongs to.
</Field>

<Field name="tab" type="string">
  The top-level category (tab) this page belongs to.
</Field>

---

## Why use `routes` instead of `allRoutes`?

`allRoutes` contains every file found by the compiler. However, if your project has multiple versions (e.g., `v1` and `v2`) or languages, you rarely want to show them all at once.

The `routes` array is intelligently pre-filtered so that when a user is browsing the **English v2** docs, it only contains other **English v2** pages.

<Tip>
  Always prefer `routes` for navigation components like Sidebars and Menus to ensure the user stays within their current context.
</Tip>
