# useConfig()

Returns the complete `BoltdocsConfig` object resolved from your `boltdocs.config.ts`. This is the same configuration object available on the server — serialized and sent to the client via a virtual module.

```tsx
import { useConfig } from 'boltdocs/client'
```

---

## Return Value

```tsx
const config: BoltdocsConfig = useConfig()

// Access theme settings
config.theme?.title          // 'My Docs'
config.theme?.navbar         // NavbarItem[]
config.theme?.editLink       // 'https://github.com/.../edit/main/docs/:path'

// Access i18n
config.i18n?.defaultLocale   // 'en'
config.i18n?.locales         // { en: 'English', es: 'Español' }

// Access versions
config.versions?.defaultVersion  // 'v2'
config.versions?.versions        // [{ label: 'v2.0', path: '2' }]

// Access integrations
config.integrations?.sandbox?.enable  // true
```

---

## Usage Example

```tsx
import { useConfig } from 'boltdocs/client'

function Footer() {
  const config = useConfig()
  const repo = config.theme?.githubRepo

  return (
    <footer>
      {repo && (
        <a href={`https://github.com/${repo}`}>
          Star us on GitHub
        </a>
      )}
    </footer>
  )
}
```

<Tip>
  The client-side config is a subset of the full Node config — it includes `theme`, `i18n`, `versions`, `integrations`, and `siteUrl`. Build-time-only fields like `docsDir` and `plugins` are not serialized.
</Tip>
