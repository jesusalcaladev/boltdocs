# useI18n()

Provides locale state and a handler for switching between languages. When the user changes locale, Boltdocs intelligently navigates to the equivalent page in the target language.

```tsx
import { useI18n } from 'boltdocs/hooks'
```

---

## Return Value

```tsx
const {
  currentLocale,        // string | undefined — e.g., 'en'
  currentLocaleLabel,   // string | undefined — e.g., 'English'
  availableLocales,     // LocaleOption[]
  handleLocaleChange,   // (locale: string) => void
} = useI18n()
```

### LocaleOption

| Property | Type | Description |
| :--- | :--- | :--- |
| `key` | `string` | Locale code (e.g., `'es'`) |
| `label` | `string` | Display name (e.g., `'Español'`) |
| `value` | `string` | Same as `key` |
| `isCurrent` | `boolean` | Whether this locale is currently active |

---

## Smart Redirection

When `handleLocaleChange` is called:

1. It finds the equivalent page in the target locale (matching by base file path)
2. If the page exists → navigates directly
3. If not → falls back to the target locale's index page
4. The selected locale is persisted in Zustand store (survives refreshes)

---

## Usage Example

```tsx
import { useI18n } from 'boltdocs/hooks'

function LocalePicker() {
  const { currentLocale, availableLocales, handleLocaleChange } = useI18n()

  return (
    <select
      value={currentLocale}
      onChange={(e) => handleLocaleChange(e.target.value)}
    >
      {availableLocales.map((locale) => (
        <option key={locale.key} value={locale.value}>
          {locale.label}
        </option>
      ))}
    </select>
  )
}
```
