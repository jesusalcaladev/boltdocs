# useSearch

`useSearch` is the hook for building custom search modals or inline search boxes. It handles the lazy loading of the FlexSearch index and manages the filtering of your documentation pages.

---

## Usage

Use `useSearch` when building a custom `SearchDialog` using the primitives.

```tsx
import { useSearch, useRoutes } from 'boltdocs/hooks'

function CustomSearch() {
  const { routes } = useRoutes()
  const { isOpen, setIsOpen, list, input } = useSearch(routes)
  
  return (
    <>
      <button onClick={() => setIsOpen(true)}>Search</button>
      
      {isOpen && (
        <div className="search-modal">
          <input {...input} />
          <ul>
            {list.map(result => (
              <li key={result.id}>
                <a href={result.path}>{result.title}</a>
              </li>
            ))}
          </ul>
        </div>
      )}
    </>
  )
}
```

---

## Return Value

| Property | Type | Description |
| :--- | :--- | :--- |
| `isOpen` | `boolean` | Whether the search interface should be visible. |
| `setIsOpen` | `fn` | Function to toggle the `isOpen` state. |
| `query` | `string` | The current search text. |
| `setQuery` | `fn` | Function to manually update the search text. |
| `list` | `SearchResult[]` | Array of filtered results matching the query. |
| `input` | `object` | Convenience props (`value`, `onChange`) for an `<input>`. |

---

## Smart Filtering

`useSearch` automatically applies context-aware filtering. It will only return results that:
1.  Match the current **Locale** (if I18n is enabled).
2.  Match the current **Version** (if versioning is enabled).
3.  Are registered in the documentation tree.

<Tip>
  The `list` returned by the hook is limited to the top 10 most relevant results to maintain high performance.
</Tip>

---

## FlexSearch Integration

The hook uses **FlexSearch** under the hood, a high-performance, client-side full-text search engine. The search index is only loaded and initialized once the search is opened for the first time, ensuring that the initial page load remains lightning-fast.
