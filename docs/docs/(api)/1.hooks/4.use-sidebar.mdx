# useSidebar

`useSidebar` is the core hook for grouping and organizing documentation pages in the sidebar. It transforms the flat `routes` array into a structured hierarchy based on your page metadata.

---

## Usage

Pass the filtered `routes` from `useRoutes` to `useSidebar` to get the grouped items.

```tsx
import { useSidebar, useRoutes } from 'boltdocs/hooks'

function CustomSidebar() {
  const { routes } = useRoutes()
  const { groups, ungrouped } = useSidebar(routes)
  
  return (
    <aside>
      {/* 1. Ungrouped (top-level) pages */}
      {ungrouped.map(route => (
        <a key={route.path} href={route.path}>{route.title}</a>
      ))}
      
      {/* 2. Grouped categories */}
      {groups.map(group => (
        <div key={group.slug}>
          <h3>{group.title}</h3>
          {group.routes.map(route => (
            <a key={route.path} href={route.path}>{route.title}</a>
          ))}
        </div>
      ))}
    </aside>
  )
}
```

---

## Return Value

| Property | Type | Description |
| :--- | :--- | :--- |
| `groups` | `SidebarGroup[]` | Pages organized into categories (groups). |
| `ungrouped` | `Route[]` | Pages that don't belong to any group. |
| `activeRoute` | `Route \| null` | The route object for the currently viewed page. |
| `activePath` | `string` | The current normalized URL path. |

---

## Group Configuration

Grouping is driven by the frontmatter and directory-level metadata. Pages with the same `group` string are grouped together.

### Example Frontmatter
```markdown
---
group: "Advanced"
groupTitle: "Advanced Guides"
groupIcon: "Settings"
---
```

<Tip>
  If `groupTitle` is missing, the `group` ID is used as the display label.
</Tip>

---

## Tab Filtering

`useSidebar` automatically filters routes based on the current active **Tab**. If the user is in the "Guides" tab, the hook will only return pages that belong to the "Guides" tab (as defined in their frontmatter or `boltdocs.config.ts`).

<Note>
  Tabs are top-level navigation categories that separate different documentation silos (e.g., Guides vs API).
</Note>
