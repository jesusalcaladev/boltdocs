# useLocalizedTo(to)

Takes a path string and returns a version of that path prefixed with the current version and locale. This ensures navigation links always stay in the correct context.

```tsx
import { useLocalizedTo } from 'boltdocs/hooks'
```

---

## Parameters

| Parameter | Type | Description |
| :--- | :--- | :--- |
| `to` | `string` | The target path (e.g., `/docs/getting-started`) |

---

## Return Value

A `string` with the appropriate `/docs/{version}/{locale}/` prefix applied.

---

## Behavior

- **External URLs** (starting with `http://` or `//`) pass through unchanged
- **Strips existing prefixes** to avoid stacking (e.g., won't double-add version)
- **Default locale is omitted** from the URL for cleaner paths
- **Non-doc links** (not starting with `/docs`) are returned as-is for non-versioned paths

---

## Usage Example

```tsx
import { useLocalizedTo } from 'boltdocs/hooks'

function MyLink() {
  // Current context: version='v2', locale='es'
  const path = useLocalizedTo('/docs/getting-started')
  // Result: '/docs/v2/es/getting-started'

  const defaultPath = useLocalizedTo('/docs/getting-started')
  // If locale is 'en' (default): '/docs/v2/getting-started'

  return <a href={path}>Go to Getting Started</a>
}
```
