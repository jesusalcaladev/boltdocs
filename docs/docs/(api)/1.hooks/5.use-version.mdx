# useVersion

`useVersion` is the hook for switching between different versions of your documentation (e.g., `v1`, `v2`, `main`). It handles URL updates and persistence when a user switches versions.

---

## Usage

Use `useVersion` to build a version switcher dropdown in your navbar.

```tsx
import { useVersion } from 'boltdocs/hooks'
import { NavbarMenu, NavbarItem } from 'boltdocs/primitives'

function VersionSwitcher() {
  const { currentVersionLabel, availableVersions, handleVersionChange } = useVersion()
  
  return (
    <NavbarMenu label={currentVersionLabel}>
      {availableVersions.map(v => (
        <NavbarItem 
          key={v.key} 
          label={v.label} 
          isCurrent={v.isCurrent}
          onPress={() => handleVersionChange(v.value)}
        />
      ))}
    </NavbarMenu>
  )
}
```

---

## Return Value

| Property | Type | Description |
| :--- | :--- | :--- |
| `currentVersion` | `string` | The active version's internal key. |
| `currentVersionLabel` | `string` | The display label for the active version. |
| `availableVersions` | `Version[]` | List of versions configured in `boltdocs.config.ts`. |
| `handleVersionChange` | `fn` | Function to switch the page to a new version. |

---

## Intelligent Switching

`handleVersionChange(version)` does more than just update the URL. When a user switches to a new version, the hook:
1.  **Tries to find the equivalent page** in the target version.
2.  If the same page exists (e.g., swapping `v1/guide/installation` for `v2/guide/installation`), it navigates directly to it.
3.  If the page does **not** exist in the new version, it redirects the user to the **index page** of that version.

<Tip>
  This intelligent switching ensures that the user doesn't hit a 404 error when exploring older or newer versions of the documentation.
</Tip>
