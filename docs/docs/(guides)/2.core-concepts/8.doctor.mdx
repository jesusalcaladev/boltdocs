# The Doctor Command

`boltdocs doctor` is a built-in health check that scans your entire documentation for common problems. Run it locally or in CI to catch issues before they reach production.

```bash
pnpm boltdocs doctor
```

---

## What It Checks

The doctor performs three categories of checks:

### 1. Broken Internal Links

Scans every markdown link (`[text](path)`) and HTML anchor (`<a href="path">`) in your documentation. For each internal link, it verifies the target file exists — checking with and without extensions (`.md`, `.mdx`, `/index.md`, `/index.mdx`).

### 2. Frontmatter Validation

- **Missing `title`**: A page without a title generates a WARNING. While Boltdocs can infer titles from filenames, explicit titles are more reliable.
- **Missing `description`**: A page without a description generates a LOW-severity notice. Descriptions improve SEO and search previews.

### 3. Orphaned Translations

When i18n is configured, the doctor checks whether every page in the default locale has a corresponding translation in all other locales. Missing translations are flagged as WARNINGs.

```text
📄 getting-started/installation.mdx
   ❌ HIGH: Broken internal link: "../api/missing-page"
      💡 Suggestion: Ensure the file exists at "docs/api/missing-page". If it's a directory, ensure it has an "index.md" or "index.mdx".

📄 core-concepts/routing.mdx
   ⚠️ WARNING: Missing "title" in frontmatter.
      💡 Suggestion: Add `title: Your Title` to the YAML frontmatter at the top of the file.
   ℹ️ LOW: Missing "description" in frontmatter.
      💡 Suggestion: Adding a description helps with SEO and search previews.

   1 high-level errors

   1 minor improvements
```

---

## Exit Codes

| Code | Meaning |
| :--- | :--- |
| `0` | All checks passed (or only warnings/low-severity issues found) |
| `1` | HIGH-severity issues detected |

---

## CI/CD Integration

Add the doctor to your CI pipeline to prevent broken documentation from being deployed:

```yaml
# GitHub Actions example
- name: Check documentation health
  run: pnpm boltdocs doctor
```

<Warning>
  The `doctor` exits with code `1` when HIGH-severity issues are found. This means your CI pipeline will fail — which is the desired behavior. You want to catch broken links before they go live.
</Warning>

<Tip>
  Run `boltdocs doctor` alongside `pnpm lint:md` for comprehensive quality assurance. The doctor checks structural issues (broken links, missing translations); markdownlint checks stylistic consistency.
</Tip>
