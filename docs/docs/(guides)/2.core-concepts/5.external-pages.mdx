# External Pages

While Boltdocs is primarily a documentation framework, you often need "external" pages that don't follow the documentation layout — like a custom Homepage, a Pricing page, or a Roadmap.

Instead of cluttering your `boltdocs.config.ts`, you can define these pages in a dedicated directory: `pages-external`.

## Getting Started

1.  Create a directory named `pages-external` inside your `docsDir` (usually the `docs/` folder).
2.  Create an `index.tsx` (or `.ts`) file inside that directory.
3.  Export your pages, custom layout, or a custom homepage from this file.

---

## Basic Example

Defining a simple Pricing page and a custom Home page.

```tsx
// docs/pages-external/index.tsx
import Pricing from './Pricing'
import Home from './Home'

/**
 * Custom Home Page (/)
 * Overrides any homePage setting in boltdocs.config.ts
 */
export const homePage = Home

/**
 * External Routes
 * Map paths to React components
 */
export const pages = {
  '/pricing': Pricing,
}
```

---

## Advanced Example: Custom Layout

You can export a `layout` component to wrap all your external pages. This is perfect for giving your marketing pages a completely different look (header, footer, etc.) from your documentation.

```tsx
// docs/pages-external/index.tsx
import Layout from './MarketingLayout'
import Pricing from './Pricing'
import Roadmap from './Roadmap'

// All pages in this module will be wrapped by this Layout
export const layout = Layout

export const pages = {
  '/pricing': Pricing,
  '/roadmap': Roadmap,
}
```

### The Layout Component

Your custom layout receives `children` and can use any Boltdocs primitives or standard React components.

```tsx
// docs/pages-external/MarketingLayout.tsx
export default function MarketingLayout({ children }: { children: React.ReactNode }) {
  return (
    <div className="marketing-wrapper">
      <nav>Custom Navigation</nav>
      <main>{children}</main>
      <footer>Custom Footer</footer>
    </div>
  )
}
```

## Technical Details

<Field name="homePage" type="React.ComponentType">
  If exported, this component will be rendered at the root path (`/`). It takes precedence over the `homePage` property in your `boltdocs.config.ts`.
</Field>

<Field name="pages" type="Record<string, React.ComponentType>">
  A map where the key is the route path (e.g., `'/pricing'`) and the value is the React component to render.
</Field>

<Field name="layout" type="React.ComponentType<{ children: React.ReactNode }>">
  An optional wrapper component for all pages defined in this module. If not provided, Boltdocs will use your project's global `layout.tsx` or the default Boltdocs layout.
</Field>

<Note>
  **Note for SEO**: All external pages are automatically included in your `sitemap.xml` and are pre-rendered during the build process (SSG) for maximum performance and SEO.
</Note>

<Tip>
  You can use standard imports relative to `docs/pages-external/index.tsx` to reach your components. We recommend putting your actual page components in the same folder or in a `src/` directory.
</Tip>
