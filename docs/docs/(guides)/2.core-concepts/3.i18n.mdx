# Internationalization (i18n)

Expanding your documentation to a global audience is effortless with Boltdocs. Instead of complex configuration files, i18n is built directly into your folder structure—making it intuitive for you and your contributors.

<Tip>
  By default, Boltdocs treats the root files as English (`en`). You can change the default locale in your `boltdocs.config.ts`.
</Tip>

---

## Folder Structure

Place your translated documents into folders named after their two-letter ISO 639-1 language codes (e.g., `es`, `fr`, `jp`). 

<FileTree>
- docs/
  - index.mdx          # Default (English) root
  - setup.mdx          # Default (English) specific path
  - es/
    - index.mdx        # Spanish route (/es)
    - setup.mdx        # Spanish setup (/es/setup)
  - fr/
    - index.mdx        # French route (/fr)
</FileTree>

---

## The Language Switcher

When Boltdocs discovers multiple locales defined via folders, it injects a **Language Switcher** component into the Navbar automatically. 

- **Smart Redirection**: When a user changes language, Boltdocs attempts to keep them on the same page (e.g., `/setup` to `/es/setup`) if it exists.
- **Fallbacks**: If a page doesn't exist in the target language, it redirects to the language's root `index.mdx`.

## Combining Versioning + i18n

If you need both Versioning and i18n, the convention is to wrap versions inside the language folders for a clean URL structure.

```text
docs/
├── en/
│   └── v1/
└── es/
    └── v1/
```

<Note>
  Need advanced custom URL mapping? Check out our [Plugins](/plugins) API for full control over the localization logic.
</Note>
