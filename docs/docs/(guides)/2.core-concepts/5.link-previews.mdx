# Link Previews

Boltdocs features a premium link preview system that provides users with a summary of the linked page when hovering over an internal link. This ensures a seamless navigation experience without needing to click through every page.

## How it Works

When you hover over any internal link (either in the sidebar or within the main content), a premium glassmorphism tooltip will appear showing the **Title** and a **Summary** of the target page.

### Page Summaries

There are two ways the preview tooltip gets its summary:

1. **Explicit Description**: Define it in the page's [Frontmatter](/docs/core-concepts/frontmatter).
2. **Automatic Extraction**: Boltdocs automatically extracts the first 160 characters and strips Markdown formatting for a clean preview.

---

## Configuration

The link preview feature is enabled by default, but you can customize it globally or per link.

### Global Toggle

<Field name="linkPreview" type="boolean" defaultValue="true">
  Update your `themeConfig` in `boltdocs.config.ts` to disable previews site-wide.
</Field>

```ts
// boltdocs.config.ts
export default {
  themeConfig: {
    linkPreview: false 
  }
}
```

### Per-Link Customization

Use the `boltdocsPreview` property if you are using the `Link` component directly in MDX to disable it for specific links.

```tsx
<Link to="/docs/install" boltdocsPreview={false}>
  Installation Guide
</Link>
```

---

## Personalization (CSS)

Customize the look of the preview tooltip using these CSS variables:

```css
:root {
  --ld-preview-bg: rgba(15, 15, 15, 0.8);
  --ld-preview-blur: 12px;
  --ld-preview-text: var(--ld-text-primary);
  --ld-preview-border: rgba(255, 255, 255, 0.1);
}
```

<Tip>
  Link previews work automatically for all local paths. External links will not show a preview tooltip.
</Tip>
