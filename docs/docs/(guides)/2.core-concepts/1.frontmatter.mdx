# Frontmatter Reference

Every `.md` and `.mdx` file in Boltdocs can have a **frontmatter** block at the very top — a section of YAML enclosed between `---` delimiters. Frontmatter lets you control how a page appears in the sidebar, what metadata gets injected into SEO tags, and how the route is generated.

```markdown
---
title: "Getting Started"
description: "Learn how to set up your first project."
icon: Rocket
badge: New
sidebarPosition: 1
---

# Getting Started

Your content here...
```

---

## Page Fields

These fields configure the appearance and behavior of individual pages.

<Field name="title" type="string">
  Override the page title. When omitted, Boltdocs infers the title from the filename by stripping numeric prefixes and extensions. For example, `2.getting-started.mdx` becomes "getting-started".
</Field>

<Field name="description" type="string">
  Used for the HTML meta description and for search result previews. If omitted, Boltdocs automatically extracts the first ~160 characters of the page content as a fallback.
</Field>

<Field name="permalink" type="string">
  Override the auto-generated URL path entirely. Useful when migrating from another docs tool and you need to preserve existing URLs for SEO.

  ```yaml
  permalink: /guides/quick-start
  ```
</Field>

<Field name="sidebarPosition" type="number">
  Explicitly control the ordering of this page within its group. Lower numbers appear first. You can also control ordering via numeric file prefixes (e.g., `1.setup.mdx`).
</Field>

<Field name="icon" type="string">
  An icon displayed next to the page title in the sidebar. Accepts any [Lucide](https://lucide.dev/icons/) icon name (e.g., `Rocket`, `Settings`, `Code`).
</Field>

<Field name="badge" type='string | { text: "new" | "updated" | "deprecated"; expires?: string }'>
  A small badge displayed next to the sidebar item. Can be a simple string or an object with an expiration date.

  ```yaml
  # Simple string
  badge: New

  # Object with auto-expiry
  badge:
    text: new
    expires: "2025-06-01"
  ```

  When `expires` is set and the date has passed, the badge is automatically hidden.
</Field>

---

## Group Fields

When used in an `index.mdx` file inside a directory, these fields control the entire group (folder) behavior.

<Field name="groupTitle" type="string">
  Override the display name of the containing directory in the sidebar. By default, Boltdocs uses the folder name (stripped of numeric prefixes and capitalized).
</Field>

<Field name="groupPosition" type="number">
  Control the ordering of the entire group relative to other groups in the sidebar.
</Field>

<Tip>
  You can combine numeric folder prefixes (e.g., `1.overview/`) with `groupPosition` in `index.mdx` for maximum control. The frontmatter value always takes priority.
</Tip>

---

## Full Example

```yaml
---
title: "Advanced Configuration"
description: "Deep-dive into every configuration option available."
icon: Wrench
badge:
  text: updated
  expires: "2025-12-31"
sidebarPosition: 10
---
```

And for a group index file (`1.getting-started/index.mdx`):

```yaml
---
title: "Welcome"
groupTitle: "Getting Started"
groupPosition: 1
icon: Sparkles
---
```
