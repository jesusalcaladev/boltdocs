# File-Based Routing

Boltdocs uses a file-based routing system built on top of React Router. Your application's routes are automatically inferred from the folder structure within the `docs/` directory.

## Basic Routing

Files are mapped automatically to URL paths based on their filename and directory. This makes organization intuitive and maintenance easy.

<FileTree>
- docs/
  - index.mdx           # -> /
  - about.mdx           # -> /about
  - api/
    - index.mdx         # -> /api
    - endpoints.mdx     # -> /api/endpoints
</FileTree>

---

## Strategic Routing

Boltdocs provides several ways to control how your routes are generated and displayed.

<Cards cols={2}>
  <Card title="Numeric Prefixes" icon={<Hash />}>
    Use numbers like `1.introduction.mdx` to force a specific sort order in the sidebar. The number is stripped from the URL.
  </Card>
  <Card title="Nested Sections" icon={<Folder />}>
    Folders with an `index.mdx` act as parent pages. Folders without one act as logical groups in the sidebar.
  </Card>
  <Card title="Custom Permalinks" icon={<Link />}>
    Override any route by adding `permalink: /my-path` to the frontmatter.
  </Card>
  <Card title="Hidden Folders" icon={<Folder />}>
    Wrap folder names in parentheses like `(guides)` to associate them with **Tabs** without affecting the URL path.
  </Card>
</Cards>

## Route Ordering Example

The visual order in your sidebar often needs to be different from alphabetical order.

```text
docs/
├── 1.getting-started/
│   ├── 1.installation.mdx    <- Served at /getting-started/installation
│   └── 2.cli.mdx             <- Served at /getting-started/cli
└── 2.core-concepts/
    └── 1.routing.mdx         <- Served at /core-concepts/routing
```

<Note>
  Notice how Boltdocs strips the numeric prefix and the `.mdx` extension to keep your URLs clean and professional.
</Note>
