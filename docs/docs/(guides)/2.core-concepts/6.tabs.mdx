# Navigation Tabs

Tabs split your documentation into high-level categories — like "Guides", "API Reference", and "Components" — displayed as a horizontal navigation bar below the main navbar.

---

## Configuration

Define tabs in your `boltdocs.config.ts`:

```ts
import { defineConfig } from 'boltdocs'

export default defineConfig({
  theme: {
    tabs: [
      { id: 'guides', text: 'Guides', icon: 'SquareLibrary' },
      { id: 'api', text: 'API', icon: 'Braces' },
      { id: 'components', text: 'Components', icon: 'Component' },
    ],
  },
})
```

<Field name="id" type="string" required>
  A unique identifier that matches a parenthesized directory name inside `docs/`. Case-insensitive — `(GUIDES)` matches id `'guides'`.
</Field>

<Field name="text" type="string | Record<string, string>" required>
  Display text for the tab. Supports i18n by passing a locale map:

  ```ts
  text: { en: 'Guides', es: 'Guías' }
  ```
</Field>

<Field name="icon" type="string">
  Optional icon displayed next to the tab text. Accepts [Lucide](https://lucide.dev/icons/) icon names.
</Field>

## Directory Structure

Associate content with a tab by wrapping the directory name in **parentheses**:

<FileTree>
- docs/
  - (guides)/
    - 1.overview/
      - 1.introduction.mdx     # Tab: guides → /docs/overview/introduction
    - 2.concepts/
      - 1.routing.mdx          # Tab: guides → /docs/concepts/routing
  - (api)/
    - hooks/
      - use-routes.mdx          # Tab: api → /docs/hooks/use-routes
  - (components)/
    - button.mdx                # Tab: components → /docs/button
</FileTree>

<Note>
  The parenthesized folder name is **stripped from the URL**. Content inside `(guides)/1.overview/` maps to `/docs/overview/`, not `/docs/guides/overview/`.
</Note>

---

## How Filtering Works

When tabs are active, the sidebar and page navigation automatically filter based on the current tab:

- **Exclusive content**: Routes inside a `(tab-id)/` directory only appear when that specific tab is selected.
- **Global content**: Files placed **outside** any tab directory are visible across **all** tabs. This is useful for shared pages like a landing page or changelog.

## The `useTabs()` Hook

Build custom tab UIs with the `useTabs()` hook:

```tsx
import { useTabs } from 'boltdocs/hooks'
import { useRoutes, useConfig } from 'boltdocs/client'

function CustomTabs() {
  const config = useConfig()
  const { routes } = useRoutes()
  const tabs = config.theme?.tabs || []

  const {
    activeIndex,       // Index of the currently active tab
    activeTabId,       // ID string of the active tab (e.g., 'guides')
    indicatorStyle,    // CSS properties for the animated underline indicator
    tabRefs,           // Ref array for measuring tab elements
  } = useTabs(tabs, routes)

  return (
    <nav>
      {tabs.map((tab, i) => (
        <a
          key={tab.id}
          ref={(el) => { tabRefs.current[i] = el }}
          className={i === activeIndex ? 'active' : ''}
        >
          {typeof tab.text === 'string' ? tab.text : tab.text.en}
        </a>
      ))}
      <div className="indicator" style={indicatorStyle} />
    </nav>
  )
}
```

## Personalization (CSS)

Customize tab appearance with these CSS variables:

```css
:root {
  --ld-tabs-bg: transparent;
  --ld-tabs-item-active-text: var(--ld-brand-primary);
  --ld-tabs-item-hover-bg: var(--ld-bg-mute);
  --ld-tabs-border: var(--ld-border-color);
}
```
