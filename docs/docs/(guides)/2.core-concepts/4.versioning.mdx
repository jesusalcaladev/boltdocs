# Documentation Versioning

Boltdocs lets you maintain multiple versions of your documentation simultaneously. Whether you're supporting legacy users on v1 or preparing a v3 release, the version switcher stays in sync with your content automatically.

---

## Configuration

Add the `versions` property to your `boltdocs.config.ts`:

```ts
import { defineConfig } from 'boltdocs'

export default defineConfig({
  versions: {
    defaultVersion: 'v2',
    prefix: 'v',
    versions: [
      { label: 'v2.0 (Latest)', path: '2' },
      { label: 'v1.0 (Legacy)', path: '1' },
    ],
  },
})
```

<Field name="defaultVersion" type="string" required>
  The version path that is active by default when no version is specified in the URL.
</Field>

<Field name="prefix" type="string">
  An optional prefix prepended to version folder names when scanning the filesystem. If set to `'v'`, a version with `path: '2'` maps to the directory `v2/`.
</Field>

<Field name="versions" type="Array<{ label: string; path: string }>" required>
  The list of versions available in the dropdown. `label` is what users see; `path` is the folder name (after the prefix) and URL segment.
</Field>

---

## Folder Structure

Create top-level directories inside `docs/` named after each version:

<FileTree>
- docs/
  - v2/
    - index.mdx                # → /docs/v2
    - getting-started.mdx      # → /docs/v2/getting-started
    - api/
      - hooks.mdx              # → /docs/v2/api/hooks
  - v1/
    - index.mdx                # → /docs/v1
    - getting-started.mdx      # → /docs/v1/getting-started
</FileTree>

<Important>
  Version directories must be at the **root** of your `docs/` folder for Boltdocs to detect them.
</Important>

## The Version Dropdown

When Boltdocs detects multiple versions, it renders a **Version Dropdown** in the navbar.

- **Intelligent switching**: When switching from `/docs/v1/getting-started` to v2, Boltdocs tries to navigate to `/docs/v2/getting-started`.
- **Graceful fallback**: If that page doesn't exist in the target version, it redirects to the version's index page.
- **Persistence**: The selected version is stored across page refreshes.

## The `useVersion()` Hook

Build custom version selectors with the `useVersion()` hook:

```tsx
import { useVersion } from 'boltdocs/hooks'

function VersionPicker() {
  const {
    currentVersion,         // 'v2'
    currentVersionLabel,    // 'v2.0 (Latest)'
    availableVersions,      // VersionOption[]
    handleVersionChange,    // (version: string) => void
  } = useVersion()

  return (
    <select
      value={currentVersion}
      onChange={(e) => handleVersionChange(e.target.value)}
    >
      {availableVersions.map((v) => (
        <option key={v.key} value={v.value}>
          {v.label}
        </option>
      ))}
    </select>
  )
}
```

Each `VersionOption` contains:

| Property | Type | Description |
| :--- | :--- | :--- |
| `key` | `string` | The version path (e.g., `'v2'`) |
| `label` | `string` | The display label (e.g., `'v2.0 (Latest)'`) |
| `value` | `string` | Same as `key`, for use in `<select>` elements |
| `isCurrent` | `boolean` | Whether this version is currently active |

## Versioning + i18n

When combining both features, versions nest **before** locales in the URL:

```
/docs/v2/es/getting-started
```

<FileTree>
- docs/
  - v2/
    - getting-started.mdx      # English v2
    - es/
      - getting-started.mdx    # Spanish v2
  - v1/
    - getting-started.mdx      # English v1
</FileTree>

<Tip>
  The `useVersion()` hook is locale-aware. When switching versions, it preserves the current locale automatically.
</Tip>
