# Documentation Versioning

Boltdocs makes it easy to maintain multiple versions of your documentation simultaneously. Whether you're supporting legacy users or preparing a major release, the version switcher stays in sync with your content automatically.

## Setting Up Versions

Versioning in Boltdocs is folder-based. Instead of having unstructured Markdown files in the `docs` directory, you place them under their respective version folders:

import { FileTree } from "boltdocs/client";

<FileTree>
 - docs/
   - v1/
     - index.mdx
     - setup.mdx
   - v2/
     - index.mdx
     - setup.mdx
</FileTree>

- Content inside `docs/v1/` will be available under `/v1/*`
- Content inside `docs/v2/` will be available under `/v2/*`

## The Version Dropdown

When Boltdocs detects top-level directories resembling versions (e.g. `v1`, `v2`, `v3.0`), it automatically groups them and generates a dropdown in the Navbar. This allows readers to seamlessly transition between API documentation versions without breaking their current reading path (if the path exists in both versions).
