# Markdown Linting

Consistent markdown styling is essential for readable documentation. Every project scaffolded with `create-boltdocs` comes pre-configured with [markdownlint-cli2](https://github.com/DavidAnson/markdownlint-cli2).

---

## Configuration

Your project includes a `.markdownlint.yaml` file at the root. Boltdocs pre-configures it to balance consistency with MDX flexibility:

```yaml
# .markdownlint.yaml
default: true
MD013: false        # Line length — disabled for long URLs and JSX props
MD033: false        # Inline HTML — disabled because MDX uses JSX extensively
MD041: false        # First line heading — disabled to allow frontmatter
MD024: false        # Duplicate headings — sometimes needed across different sections
```

The `.markdownlintignore` file excludes directories that shouldn't be linted:

```text
node_modules/
dist/
.boltdocs/
```

## Customizing Rules

Open `.markdownlint.yaml` and toggle any [markdownlint rule](https://github.com/DavidAnson/markdownlint/blob/main/doc/Rules.md):

```yaml
# Enable line length enforcement (120 char max)
MD013:
  line_length: 120
  tables: false

# Enable duplicate heading detection
MD024: true
```

---

## Editor Integration

We recommend installing the markdownlint extension for real-time feedback:

- **VS Code**: Install [`DavidAnson.vscode-markdownlint`](https://marketplace.visualstudio.com/items?itemName=DavidAnson.vscode-markdownlint)
- **IntelliJ**: Install the **Markdownlint** plugin from the JetBrains marketplace

<Tip>
  Pair markdownlint with `boltdocs doctor` for complete documentation quality coverage. Markdownlint checks style; doctor checks broken links, missing metadata, and orphaned translations.
</Tip>
