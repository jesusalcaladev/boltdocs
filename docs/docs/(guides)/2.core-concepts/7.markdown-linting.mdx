# Markdown Linting

Maintaining consistent markdown styling is essential for clean documentation. By default, every new project created with `create-boltdocs` comes fully configured with [`markdownlint-cli2`](https://github.com/DavidAnson/markdownlint-cli2) right out of the box.

## Configuration

Your generated project includes a `.markdownlint.yaml` file at the root. We've pre-configured several rules to balance consistency with MDX flexibility.

<Note>
  We disabled rules like **MD013 (Line Length)** and **MD033 (Inline HTML)** because documentation often requires long URLs or JSX syntax which would otherwise trigger unnecessary errors.
</Note>

## Customizing Rules

Because the configuration lives directly in your project's root folder, you have full control. To adjust a rule, open `.markdownlint.yaml` and toggle the boolean flags.

<Tip>
  Reference the [Official Rules Documentation](https://github.com/DavidAnson/markdownlint/blob/main/doc/Rules.md) for a complete list of available linting options.
</Tip>

### Editor Integration

We highly recommend installing the **markdownlint** extension for your editor to see feedback in real-time.

- **VS Code**: Install `DavidAnson.vscode-markdownlint`.
- **IntelliJ**: Install the **Markdownlint** plugin from the marketplace.
