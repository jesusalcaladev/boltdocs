# Custom Layout

Boltdocs ships with a `DefaultLayout` that arranges the navbar, sidebar, content area, table of contents, breadcrumbs, and page navigation. If the default arrangement doesn't fit your needs, you can create a `layout.tsx` file that gives you full control over every section.

---

## How It Works

Boltdocs uses a virtual module (`virtual:boltdocs-layout`) to resolve the layout component. The resolution order is:

1. **Your `layout.tsx`** — If `docs/layout.tsx` (or `.jsx`) exists, Boltdocs uses it.
2. **`DefaultLayout`** — If no custom layout exists, the built-in layout is used.

<Note>
  Changes to `layout.tsx` are picked up automatically by HMR. Save the file and the browser updates instantly.
</Note>

---

## Creating a Custom Layout

Create a `layout.tsx` file inside your `docs/` directory:

<FileTree>
- docs/
  - layout.tsx             # Your custom layout
  - mdx-components.tsx     # Optional MDX overrides
  - (guides)/
    - ...
</FileTree>

Your layout must be a React component that receives `children` (the rendered MDX content):

```tsx
// docs/layout.tsx
export default function Layout({ children }: { children: React.ReactNode }) {
  return (
    <div>
      {/* Your custom layout here */}
      {children}
    </div>
  )
}
```

## Building Blocks

Boltdocs exports composable layout components from `boltdocs/client` that you can arrange however you want:

### `DocsLayout` (Compound Component)

The root layout shell. It provides the structural containers:

| Slot | Purpose |
| :--- | :--- |
| `<DocsLayout>` | Root wrapper — full-height flex column with theme-aware background |
| `<DocsLayout.Body>` | Horizontal container for sidebar + content + ToC |
| `<DocsLayout.Content>` | Main scrollable content area with max-width constraint |
| `<DocsLayout.ContentHeader>` | Header row inside content (breadcrumbs, copy button) |
| `<DocsLayout.ContentFooter>` | Footer row inside content (page navigation) |

Each slot accepts `children`, `className`, and `style` props for customization.

### UI Components

| Component | Import | Purpose |
| :--- | :--- | :--- |
| `Navbar` | `boltdocs/client` | Full-featured top navigation with logo, links, search, theme toggle |
| `Sidebar` | `boltdocs/client` | Left navigation with groups, badges, icons, version/locale switchers |
| `OnThisPage` | `boltdocs/client` | Right-side table of contents with heading tracking |
| `Head` | `boltdocs/client` | SEO meta tags (`<title>`, description, Open Graph) |
| `Breadcrumbs` | `boltdocs/client` | Breadcrumb navigation based on current route |
| `PageNav` | `boltdocs/client` | Previous / Next page navigation |
| `ProgressBar` | `boltdocs/client` | Reading progress indicator at the top of the page |
| `ErrorBoundary` | `boltdocs/client` | Catches rendering errors and shows a friendly fallback |
| `CopyMarkdown` | `boltdocs/client` | Button to copy the raw MDX content |

### Key Hooks

| Hook | Purpose |
| :--- | :--- |
| `useRoutes()` | Get filtered routes, current route, locale, version |
| `useConfig()` | Access the site configuration |
| `useLocation()` | Get the current URL pathname |
| `useMdxComponents()` | Access registered MDX component overrides |

## Full Example

Here's the complete layout used by the Boltdocs documentation site itself:

```tsx
// docs/layout.tsx
import {
  DocsLayout,
  Navbar,
  Sidebar,
  OnThisPage,
  Head,
  Breadcrumbs,
  PageNav,
  ProgressBar,
  ErrorBoundary,
  CopyMarkdown,
  useRoutes,
  useConfig,
  useMdxComponents,
} from 'boltdocs/client'
import { useLocation } from 'boltdocs/hooks'

export default function Layout({ children }: { children: React.ReactNode }) {
  const { routes: filteredRoutes, allRoutes, currentRoute } = useRoutes()
  const { pathname } = useLocation()
  const config = useConfig()
  const mdxComponents = useMdxComponents()

  // Allow CopyMarkdown override via mdx-components.tsx
  const CopyMarkdownComp = (mdxComponents.CopyMarkdown as any) || CopyMarkdown

  const isHome = pathname === '/' || pathname === ''

  return (
    <DocsLayout>
      <ProgressBar />
      <Head
        siteTitle={config.theme?.title || 'Boltdocs'}
        siteDescription={config.theme?.description || ''}
        routes={allRoutes}
      />
      <Navbar />

      <DocsLayout.Body>
        {!isHome && <Sidebar routes={filteredRoutes} config={config} />}

        <DocsLayout.Content>
          {!isHome && (
            <DocsLayout.ContentHeader>
              <Breadcrumbs />
              <CopyMarkdownComp
                mdxRaw={currentRoute?._rawContent}
                route={currentRoute}
                config={config.theme?.copyMarkdown}
              />
            </DocsLayout.ContentHeader>
          )}

          <ErrorBoundary>{children}</ErrorBoundary>

          {!isHome && (
            <DocsLayout.ContentFooter>
              <PageNav />
            </DocsLayout.ContentFooter>
          )}
        </DocsLayout.Content>

        {!isHome && (
          <OnThisPage
            headings={currentRoute?.headings}
            editLink={config.theme?.editLink}
            communityHelp={config.theme?.communityHelp}
            filePath={currentRoute?.filePath}
          />
        )}
      </DocsLayout.Body>
    </DocsLayout>
  )
}
```

## Component Props Reference

### Sidebar

<Field name="routes" type="ComponentRoute[]" required>
  The filtered routes to display in the sidebar (usually from `useRoutes().routes`).
</Field>

<Field name="config" type="BoltdocsConfig" required>
  The site configuration object (from `useConfig()`).
</Field>

### OnThisPage

<Field name="headings" type="Array<{ level: number; text: string; id: string }>">
  The extracted headings from the current page.
</Field>

<Field name="editLink" type="string">
  URL template for the "Edit this page" link. Uses `:path` as a placeholder.
</Field>

<Field name="communityHelp" type="string">
  URL for the "Get help" link.
</Field>

<Field name="filePath" type="string">
  The relative file path of the current page (used to construct the edit link).
</Field>

### Head

<Field name="siteTitle" type="string" required>
  The base title for the `<title>` tag and OG metadata.
</Field>

<Field name="siteDescription" type="string" required>
  The default description for meta tags.
</Field>

<Field name="routes" type="ComponentRoute[]" required>
  All routes (used to generate canonical URLs).
</Field>

<Tip>
  If CSS variable overrides and layout composition still aren't enough, check the [Primitives tab](/docs/primitives/navbar) for individually composable building blocks you can style from scratch.
</Tip>
