# MDX Components

Boltdocs allows you to override the default HTML elements rendered by MDX **and** register custom React components available globally in all your `.mdx` files — no imports needed.

---

## The `mdx-components.tsx` File

Create this file in your **documentation directory** (e.g., `docs/mdx-components.tsx`). Boltdocs automatically discovers it through a virtual module (`virtual:boltdocs-mdx-components`) and passes its exports to the MDX provider.

Supported extensions: `.tsx`, `.ts`, `.jsx`, `.js`

### Basic Setup

Export a default object where keys are component names or HTML tags:

```tsx
// docs/mdx-components.tsx

export default {
  // Override a standard HTML element
  h1: (props: any) => (
    <h1 style={{ color: '#7f13ec' }} {...props} />
  ),

  // Register a custom component
  MyBadge: ({ children }: { children: React.ReactNode }) => (
    <span className="bg-purple-100 text-purple-800 px-2 py-1 rounded-md text-sm">
      {children}
    </span>
  ),
}
```

---

## Overriding HTML Elements

Override any standard HTML tag generated by the MDX compiler:

| Tag | Common Use Case |
| :--- | :--- |
| `h1`–`h6` | Custom heading styles, copyable anchor links |
| `pre` | Custom code blocks with syntax highlighting |
| `a` | Use `react-router-dom` for internal links, external link indicators |
| `img` | Lazy loading, theme-aware image variants |
| `table` | Custom styled tables |

### Example: Linked Headings

```tsx
export default {
  h2: ({ children, id }: any) => (
    <h2 id={id} className="group flex items-center gap-2">
      {children}
      <a
        href={`#${id}`}
        className="opacity-0 group-hover:opacity-100 transition-opacity text-primary-500"
      >
        #
      </a>
    </h2>
  ),
}
```

## Global Custom Components

Components registered in `mdx-components.tsx` can be used in **any** MDX file without importing them:

### In `mdx-components.tsx`:

```tsx
export default {
  Callout: ({ title, children }: { title: string; children: React.ReactNode }) => (
    <div className="border-l-4 border-primary-500 bg-primary-500/10 p-4 rounded-r-lg my-4">
      <p className="font-semibold text-primary-500 mb-1">{title}</p>
      {children}
    </div>
  ),
}
```

### In any `.mdx` file (no import needed):

```mdx
# Welcome

<Callout title="Quick Tip">
  Components defined in mdx-components.tsx are available everywhere!
</Callout>
```

## Overriding CopyMarkdown

The `CopyMarkdown` button at the top of each page can be replaced through `mdx-components.tsx`:

```tsx
// docs/mdx-components.tsx

function MyCopyButton({ mdxRaw, route }: any) {
  return (
    <button onClick={() => navigator.clipboard.writeText(mdxRaw || '')}>
      📋 Copy Source
    </button>
  )
}

export default {
  CopyMarkdown: MyCopyButton,
}
```

The component automatically receives `mdxRaw` (the raw markdown content) and `route` (the current route metadata) as props.

## Accessing Boltdocs Context

Because MDX components render inside the standard providers, you can use all Boltdocs hooks:

```tsx
import { useConfig, useTheme } from 'boltdocs/client'

export default {
  ThemeInfo: () => {
    const config = useConfig()
    const { theme } = useTheme()
    return <p>Current theme: {theme}, Site: {config.theme?.title}</p>
  },
}
```

## The `useMdxComponents()` Hook

Access the registered MDX components programmatically in your layout or custom components:

```tsx
import { useMdxComponents } from 'boltdocs/client'

function MyLayout({ children }: { children: React.ReactNode }) {
  const components = useMdxComponents()
  const CopyBtn = (components.CopyMarkdown as any) || DefaultCopyMarkdown
  // ...
}
```

---

## Best Practices

- **Keep it lightweight**: These components are injected globally — heavy components will affect all pages.
- **Use TypeScript**: Define prop interfaces for better IDE support inside MDX files.
- **Consistent design**: Use `mdx-components.tsx` to enforce brand typography and spacing across all elements.

<Tip>
  Need to swap entire layout sections? Use a custom [layout.tsx](/docs/guides/customization/layout) instead of MDX component overrides.
</Tip>
