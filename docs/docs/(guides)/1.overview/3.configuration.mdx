Boltdocs is designed to be flexible and dev-friendly. Everything that makes your documentation site unique—from your logo and site title to advanced search engine optimization and multi-language support—is controlled through a central configuration file.

## Configuration Files

Boltdocs automatically detects your settings by looking for one of these files in your project's root:

- `boltdocs.config.ts` (Recommended for the best developer experience)
- `boltdocs.config.js`
- `boltdocs.config.mjs`

Using **TypeScript** is highly encouraged. It provides instant autocomplete, property validation, and serves as living documentation while you type.

## Basic Example

Getting started? A minimal configuration is all you need to go live.

```ts
import { defineConfig } from 'boltdocs';

export default defineConfig({
  theme: {
    title: 'My Project',
    description: 'Documentation made simple.',
    navbar: [
      { label: 'Guides', href: '/docs' },
      { label: 'GitHub', href: 'https://github.com/user/repo' }
    ]
  }
});
```

## Advanced Example

For production-ready sites, you can leverage granular SEO, internationalization, and customized code themes.

```ts
import { defineConfig } from 'boltdocs';

export default defineConfig({
  siteUrl: 'https://docs.my-project.com',
  docsDir: 'docs',
  theme: {
    title: 'Boltdocs Pro',
    logo: {
      dark: '/logo-dark.png',
      light: '/logo-light.png',
      alt: 'Boltdocs Logo'
    },
    socialLinks: [
      { icon: 'github', link: 'https://github.com/boltdocs' },
      { icon: 'x', link: 'https://x.com/boltdocs' }
    ],
    codeTheme: {
      light: 'github-light',
      dark: 'tokyo-night' // Custom Shiki theme
    },
    sidebarGroups: {
      'overview': { title: 'Introduction', icon: 'book' },
      'api': { title: 'API Reference', icon: 'code' }
    }
  },
  seo: {
    metatags: {
      'twitter:site': '@boltdocs',
      'og:site_name': 'Boltdocs Official Docs'
    },
    indexing: 'all'
  },
  i18n: {
    defaultLocale: 'en',
    locales: {
      en: 'English',
      es: 'Español'
    }
  }
});
```

## API Reference

A deep dive into every property available within the `defineConfig` helper.

### Root Properties

| Property | Type | Description |
| :--- | :--- | :--- |
| `siteUrl` | `string` | The absolute URL of your site (e.g., `https://docs.com`). Required for Sitemap generation. |
| `docsDir` | `string` | Relative path to your documentation folder. Defaults to `docs`. |
| `base` | `string` | The base path of your application (e.g., `/docs/`). Useful if your site is not at the domain root. |
| `homePage` | `string` | Path to the file that will serve as your landing page. |
| `theme` | `BoltdocsThemeConfig` | Aesthetics, navigation, and identity settings. |
| `seo` | `BoltdocsSeoConfig` | Global meta tags and search engine crawler instructions. |
| `i18n` | `BoltdocsI18nConfig` | Multi-language and localization settings. |
| `versions` | `BoltdocsVersionsConfig` | Documentation versioning configuration. |
| `plugins` | `BoltdocsPlugin[]` | List of active framework plugins. |
| `robots` | `BoltdocsRobotsConfig` | Manual configuration for your `robots.txt`. |
| `security` | `BoltdocsSecurityConfig` | Security headers and Basic CSP settings. |
| `vite` | `any` | Advanced: Options to pass directly to the underlying Vite instance. |

### Theme Configuration (`theme`)

Shapes the visual identity and navigation flow of your documentation.

| Property | Type | Description |
| :--- | :--- | :--- |
| `title` | `string \| Record` | Site title. Supports strings or i18n objects. |
| `description` | `string \| Record` | General site description for footer and SEO. |
| `logo` | `string \| object` | Logo path. Can be a string or `{ dark, light, alt, width, height }`. |
| `favicon` | `string` | Path to your site's favicon. |
| `ogImage` | `string` | Default image for Social Sharing (Open Graph). |
| `navbar` | `Array` | Links for the top navigation bar: `{ label, href }`. |
| `sidebar` | `Record` | Manual sidebar configuration (if not using auto-discovery). |
| `sidebarGroups` | `Record` | Customize folder titles and icons in the sidebar: `{ title, icon }`. |
| `socialLinks` | `Array` | Social media icons: `{ icon, link }`. Supported: `github`, `discord`, `x`, `bluesky`. |
| `footer` | `object` | Text displayed at the very bottom of each page: `{ text }`. |
| `breadcrumbs` | `boolean` | Toggle the page path indicator. Defaults to `true`. |
| `codeTheme` | `string \| object` | Shiki highlight theme. e.g., `github-dark` or `{ light: '...', dark: '...' }`. |
| `copyMarkdown` | `boolean \| object` | Enables a button to copy the raw Markdown content. |
| `githubRepo` | `string` | Link to your source repo. Enables "Edit this page" buttons. |
| `editLink` | `string` | Pattern for the edit link (e.g., `https://github.com/.../edit/main/:path`). |
| `communityHelp` | `string` | Link to your community support (Discord, Forum, etc.). |
| `version` | `string` | Static version tag displayed in the sidebar/footer. |
| `tabs` | `Array` | Top-level tabs for grouping broadly different content: `{ id, text, icon }`. |
| `poweredBy` | `boolean` | Display the subtle "Powered by Boltdocs" branding. |

### SEO (`seo`)

Take control of how search engines and social platforms index and display your content.

| Property | Type | Description |
| :--- | :--- | :--- |
| `metatags` | `Record<string, string>` | Global meta tags (e.g., `twitter:creator`). |
| `indexing` | `'all' \| 'public'` | Defines the global indexing policy. |
| `thumbnails` | `object` | Dynamic OG image background settings. |

#### Example
```ts
seo: {
  metatags: {
    // Open Graph Prefixes (og:)
    'og:title': 'My Documentation Title',
    'og:description': 'A detailed site description.',
    'og:image': '/social-preview.png',
    
    // Twitter Card Prefixes (twitter:)
    'twitter:card': 'summary_large_image',
    'twitter:site': '@boltdocs',
    
    // Meta name tags
    'author': 'Boltdocs Team',
    'keywords': 'documentation, react, vite',
    'robots': 'index, follow',
    'theme-color': '#0ea5e9',
    
    // Apple / Mobile specific
    'apple-mobile-web-app-capable': 'yes',
    'viewport': 'width=device-width, initial-scale=1'
  },
  indexing: 'all',
  thumbnails: {
    background: '/images/og-background.png'
  }
}
```

### Internationalization (`i18n`)

| Property | Type | Description |
| :--- | :--- | :--- |
| `defaultLocale` | `string` | The primary language prefix (e.g., `en`). |
| `locales` | `Record<string, string>` | Mapping of prefixes to names: `{ en: 'English', es: 'Español' }`. |
| `localeConfigs` | `Record` | Language-specific settings: `{ direction: 'ltr' \| 'rtl', htmlLang: 'en-US' }`. |

#### Example
```ts
i18n: {
  defaultLocale: 'en',
  locales: {
    en: 'English',
    es: 'Español'
  },
  localeConfigs: {
    es: {
      label: 'Español',
      direction: 'ltr',
      htmlLang: 'es-ES'
    },
    en: {
      label: 'United States',
      htmlLang: 'en-US'
    }
  }
}
```

### Versioning (`versions`)

| Property | Type | Description |
| :--- | :--- | :--- |
| `defaultVersion` | `string` | The version shown by default to visitors. |
| `prefix` | `string` | Prefix for version folders (e.g., `v`). |
| `versions` | `Array` | List of available documentation versions: `{ label, path }`. |

#### Example
```ts
versions: {
  defaultVersion: '2.0.0',
  prefix: 'v',
  versions: [
    { label: 'Version 2.0 (Stable)', path: '2.0.0' },
    { label: 'Version 1.5', path: '1.5.0' },
    { label: 'Legacy (v0.9)', path: '0.9.0' }
  ]
}
```

### Security (`security`)

| Property | Type | Description |
| :--- | :--- | :--- |
| `enableCSP` | `boolean` | Enables basic Content-Security-Policy headers. |
| `headers` | `Record` | Custom HTTP headers (applied during production build). |
| `customHeaders` | `Record` | Alias for additional header injection. |
