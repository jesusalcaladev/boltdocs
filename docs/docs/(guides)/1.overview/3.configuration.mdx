# Configuration

The `boltdocs.config.js` (or `.ts`, `.mjs`) file at the root of your project controls how Boltdocs functions, builds routes, and renders the UI.

<Tip>
  We strongly recommend adding a JSDoc type annotation for autocomplete and type safety.
</Tip>

```js
/**
 * @type {import('boltdocs').BoltdocsConfig}
 */
export default {
  // Your options...
};
```

---

## Root Options (`BoltdocsConfig`)

### `siteUrl`
<Field name="siteUrl" type="string">
The base URL of the site, used for generating absolute paths in your sitemaps and canonical URLs.
</Field>

### `docsDir`
<Field name="docsDir" type="string" defaultValue='"docs"'>
The root directory containing your Markdown/MDX documentation files. 
</Field>

<Important title="Theme Config">
  The `themeConfig` property is **required** and holds the majority of your site's visual and functional settings.
</Important>

### `themeConfig`
<Field name="themeConfig" type="BoltdocsThemeConfig" required>
  See [Theme Configuration](#theme-configuration) for a full breakdown.
</Field>

### `i18n`
<Field name="i18n" type="BoltdocsI18nConfig">
  Configuration for internationalization.
  - `defaultLocale` (`string`): The fallback locale (e.g., `"en"`).
  - `locales` (`Record<string, string>`): Available locales mapping.
</Field>

### `versions`
<Field name="versions" type="BoltdocsVersionsConfig">
  Configuration for documentation versioning.
  - `defaultVersion` (`string`): The fallback version (e.g., `"v2"`).
  - `versions` (`Record<string, string>`): Available versions mapping.
</Field>

## Theme Configuration

### `title`
<Field name="title" type="string" defaultValue='"Boltdocs"'>
  The global title of your documentation site, displayed in the navbar and used for SEO.
</Field>

### `description`
<Field name="description" type="string">
  The global site description, used for SEO meta tags.
</Field>

### `logo`
<Field name="logo" type="string | object">
  The URL path to your site logo. Supports a simple string path or a specialized object for light/dark versions.
</Field>

```ts
// boltdocs.config.ts
export default {
  themeConfig: {
    // Basic format
    logo: "/logo.png",

    // Advanced themed format
    logo: {
      dark: "/logo-dark.png",
      light: "/logo-light.png",
      alt: "LiteDocs",
      width: 32,
      height: 32
    }
  }
}
```

### `navbar`
<Field name="navbar" type="NavbarItem[]">
  Items to display in the top navigation bar.
  - `text` / `label` (`string`): Display text.
  - `link` / `href` (`string`): Destination URL.
  - `position` (`"left" | "right"`): Alignment.
</Field>

### `sidebar`
<Field name="sidebar" type="SidebarConfig">
  Custom sidebar groups. While Boltdocs generates the sidebar automatically, you can override it here.
</Field>

### `githubRepo`
<Field name="githubRepo" type="string">
  Points to your GitHub repo in `owner/repo` format. Enables the GitHub stars badge in the navbar.
</Field>

### `editLink`
<Field name="editLink" type="string">
  URL template for the 'Edit this page' link. Use `:path` for the current file.
</Field>

## Personalization (CSS)

Boltdocs uses CSS variables (tokens) for all styling. You can override these in your `custom.css` file.

```css
:root {
  /* Brand Colors */
  --ld-brand-primary: #7f13ec;
  --ld-brand-secondary: #ff4d4d;

  /* Backgrounds */
  --ld-bg-main: #ffffff;
  --ld-bg-soft: #f9f9f9;
  --ld-bg-mute: #f1f1f1;

  /* Typography */
  --ld-text-main: #1a1a1a;
  --ld-text-muted: #666666;
}
```

## Vite Plugin Options

In your `vite.config.ts`, you can pass options directly to the `boltdocs()` plugin.

```ts
export default defineConfig({
  plugins: [
    boltdocs({
      docsDir: "./docs",
      homePage: "./src/HomePage.tsx",
    })
  ]
});
```

<Field name="homePage" type="string">
  Path to a custom React component to render at the base route `/`.
</Field>
