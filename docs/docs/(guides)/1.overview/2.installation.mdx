# Installation

Get up and running with Boltdocs in under a minute. Our scaffolding tool handles the heavy lifting so you can focus on writing.

<Note title="Requirements">
  Boltdocs requires **Node.js 18.0.0** or newer. We recommend using **pnpm** for the best experience.
</Note>

## Quick Start

Follow these simple steps to bootstrap your documentation project.

### 1. Bootstrap your project

Run the following command to kickstart your new documentation site:

```bash
pnpm create boltdocs@latest
```

<Tip>
  The CLI will ask for a **Project Name** and a **Template**. We recommend starting with the **Base Template** as it includes a custom homepage and example components.
</Tip>

### 2. Launch the dev server

Navigate to your new folder and fire up the development environment:

```bash
cd my-boltdocs-site
pnpm install
pnpm run dev
```

Your site is now live at `http://localhost:5173`. Any changes you make to your Markdown files will reflect instantly!

---

## Project Structure

After installation, your project will follow this standard structure:

<FileTree>
- docs/
  - index.mdx
  - (guides)/
    - 1.overview.mdx
- src/
  - components/
- boltdocs.config.js
- package.json
- custom.css (optional)
- vite.config.ts
</FileTree>

## Production Build

When your docs are ready for the world, generate a highly optimized static build:

<Cards cols={2}>
  <Card title="Build" icon="🚀">
    `pnpm run build`
    Generates production files in the `dist/` directory.
  </Card>
  <Card title="Preview" icon="👀">
    `pnpm run preview`
    Spin up a local server to test the production build.
  </Card>
</Cards>
