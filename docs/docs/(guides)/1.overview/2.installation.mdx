# Installation

Get up and running with Boltdocs in under a minute. Our scaffolding tool handles the heavy lifting so you can focus on writing.

### Prerequisites

Ensure you have **Node.js 18 or newer** installed on your machine.

## Quick Start

We recommend using `pnpm` for its speed and efficiency, but `npm` or `yarn` work perfectly too.

### 1. Bootstrap your project

Run the following command to kickstart your new documentation site:

```bash
pnpm create boltdocs@latest
```

The CLI will guide you through a couple of quick choices:
- **Project Name**: Choose a name for your new directory.
- **Template**: 
    - **Base Template (Recommended)**: A complete setup with a custom homepage and example components.
    - **Empty Template**: A clean slate with only the bare essentials.

### 2. Install and launch

Navigate to your new folder, install the dependencies, and fire up the development server:

```bash
cd my-boltdocs-site
pnpm install
pnpm run dev
```

Your site is now live at `http://localhost:5173`. Any changes you make to your Markdown files will reflect instantly!

## Preparing for Production

When your docs are ready for the world, generate a highly optimized static build:

```bash
pnpm run build
```

The production-ready files will be waiting in the `dist/` directory. You can preview the final result locally with:

```bash
pnpm run preview
```
