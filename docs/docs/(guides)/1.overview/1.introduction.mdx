# What is Boltdocs?

Boltdocs is a documentation framework built directly on top of **Vite** and **React**. You write your docs in **MDX** (Markdown with JSX), and Boltdocs turns them into a fast, beautiful, and fully searchable documentation site — with zero boilerplate configuration.

If you've ever struggled with documentation tools that fight you on customization, Boltdocs is the opposite. Every layer of the framework is designed to be overridden — from individual CSS variables all the way down to replacing entire layout sections with your own React components.

## How It Works

At its core, Boltdocs is a **Vite plugin** that:

1. **Scans** your `docs/` directory for `.md` and `.mdx` files
2. **Parses** frontmatter, headings, and content from each file
3. **Generates** routes automatically based on the folder structure
4. **Renders** everything through a React app with built-in navigation, search, and theming

You don't need to configure a router, set up a build pipeline, or wire up a search index. Boltdocs does it all — and gives you the hooks to customize every piece of it.

<Tip>
  Boltdocs is "just Vite" under the hood. Any Vite plugin, Tailwind setup, or React pattern you already know will work seamlessly.
</Tip>

## Architecture Overview

Boltdocs is split into two main layers:

| Layer | Package | What It Does |
| :--- | :--- | :--- |
| **Node** | `boltdocs` | Config resolution, file-system routing, MDX compilation, SSG, CLI, plugin orchestration |
| **Client** | `boltdocs/client` | React components, hooks, layout primitives, theme system, search, state management |

Additional entry points give you granular access:

| Import | Purpose |
| :--- | :--- |
| `boltdocs/hooks` | All React hooks (`useRoutes`, `useI18n`, `useVersion`, etc.) |
| `boltdocs/primitives` | Headless UI building blocks (Navbar, Sidebar, Popover, Tooltip, etc.) |
| `boltdocs/mdx` | MDX components (Button, Card, Admonition, Table, etc.) |
| `boltdocs/integrations` | Third-party integrations (CodeSandbox) |
