# Command Line Interface (CLI)

Boltdocs comes with a powerfull scaffolding tool named `create-boltdocs` that helps you bootstrap a new documentation site in seconds.

## Usage

Run the following command in your terminal to get started:

```bash
npx create-boltdocs@latest
```

## Commands & Options

| Command | Description |
| :--- | :--- |
| `pnpm create boltdocs` | Interactive wizard to bootstrap a new project. |
| `--template <name>` | Skip the wizard and use a specific template (`base` or `empty`). |
| `--help` | Show all available CLI flags and options. |

---

## What it generates

When you run the CLI, it creates a fully configured documentation environment. Here is a look at the generated project structure:

<FileTree>
- my-docs/
  - docs/
    - index.mdx          # Your entry point
  - src/
    - components/       # Custom React components
  - boltdocs.config.js  # Theme configuration
  - custom.css          # Your design tokens
  - package.json        # Dependencies
  - vite.config.ts      # Vite & Plugin setup
</FileTree>

## Customization

Once the CLI finishes, you own the code! Because Boltdocs is built on **Vite**, you can:

- **Add Plugins**: Install any Vite, Remark, or Rehype plugins.
- **Use Tailwind**: Configure Tailwind CSS or Sass as you would in any Vite project.
- **Custom Components**: Import your own React components directly into MDX.

<Note>
  Boltdocs stays out of your way. Anything you can do with Vite, you can do with Boltdocs.
</Note>
