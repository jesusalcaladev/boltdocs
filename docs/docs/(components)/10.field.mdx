# Field

The Field component is designed to provide a structured way to document individual API properties, configuration options, or any specific data field. 

<Warning>
  **Legacy Status**: While `<Field />` is fully supported, we recommend using the new, more structured `<ComponentProps /` component for documenting complete sets of props in a table format.
</Warning>

## Overview

<Field 
  name="siteUrl" 
  type="string" 
  required
  defaultValue='"https://boltdocs.dev"'
>
  The base URL of your documentation site. It is primarily used for generating canonical URLs and absolute paths in your sitemap.
</Field>

## Variants & Examples

### Basic Usage

Use `<Field>` for standalone definitions where a full table might be unnecessary.

<Field name="title" type="string">
  The title text displayed in the navigation bar.
</Field>

### Complex Types

Fields can document complex configuration objects by using the `type` prop for interface names.

<Field 
  name="themeConfig" 
  type="BoltdocsThemeConfig" 
  required
>
  The central configuration object defining the visual style and navigation structure of your site.
</Field>

### Personalization (CSS)

Adjust the background, borders, and colors of the field component via these CSS variables in your `custom.css`.

```css
:root {
  /* Container Appearance */
  --ld-field-bg: var(--ld-bg-soft);
  --ld-field-border: var(--ld-border-subtle);
  
  /* Text & Labels */
  --ld-field-name-color: var(--ld-color-primary);
  --ld-field-type-color: var(--ld-text-muted);
  
  /* Required Badge */
  --ld-field-required-bg: rgba(239, 68, 68, 0.15);
  --ld-field-required-text: #f87171;
}
```
