# Component Props

The ComponentProps component is a high-performance, aesthetically pleasing UI element designed specifically for documenting component APIs. It provides a structured grid layout that makes properties, types, and default values easy to scan and read.

## Overview

<ComponentProps
  title="Button Property Reference"
  props={[
    {
      name: "variant",
      type: "'primary' | 'secondary' | 'danger'",
      defaultValue: "'primary'",
      description: "The visual style of the button."
    },
    {
      name: "size",
      type: "'sm' | 'md' | 'lg'",
      defaultValue: "'md'",
      description: "The scale of the button component."
    },
    {
      name: "disabled",
      type: "boolean",
      defaultValue: "false",
      description: "When true, disables all interactions."
    }
  ]}
/>

## Variants & Examples

### PropItem Definition

Each object in the `props` array should follow this interface:

<ComponentProps
  props={[
    {
      name: "name",
      type: "string",
      required: true,
      description: "The name of the property."
    },
    {
      name: "type",
      type: "string",
      required: true,
      description: "The data type string or union (e.g., 'boolean', 'string')."
    },
    {
      name: "defaultValue",
      type: "string",
      description: "The value used if the property is omitted."
    },
    {
      name: "required",
      type: "boolean",
      defaultValue: "false",
      description: "If true, displays a red '*' or badge next to the name."
    },
    {
      name: "description",
      type: "ReactNode",
      required: true,
      description: "A detailed explanation of the property's role."
    }
  ]}
/>

### Rich Descriptions

The `description` field supports React nodes, allowing you to include links, bold text, or even other components for detailed explanations.

<ComponentProps
  title="Plugin Configuration"
  props={[
    {
      name: "options",
      type: "PluginOptions",
      description: (
        <span>
          Configuration object. See the <Link to="/docs/plugins/overview">Plugin Guide</Link> for more details.
        </span>
      )
    }
  ]}
/>
