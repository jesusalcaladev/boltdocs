# Link

The `<Link />` component is the standard way to navigate between pages in Boltdocs. It automatically distinguishes between internal documentation routes and external web links, providing a premium experience with support for **Link Previews**.

## Usage

Use `<Link />` for both internal navigation and external links. 

<Playground 
  code={`
import { Link, Note } from 'boltdocs/client';

export default function Demo() {
  return (
    <div style={{ padding: '20px' }}>
      <Note title="Navigation Example">
        Clicking the first link will use SPA routing. The second will open in a new tab.
      </Note>
      <ul style={{ marginTop: '16px', display: 'flex', flexDirection: 'column', gap: '8px' }}>
        <li>
          <Link to="/docs/overview/introduction">Internal: Introduction</Link>
        </li>
        <li>
          <Link to="https://github.com/jesusalcaladev/litedocs">External: GitHub Repo</Link>
        </li>
      </ul>
    </div>
  );
}
`} />

---

## Technical Features

- **SEO Friendly**: Uses standard `<a>` tags under the hood for search engine crawlability.
- **SPA Aware**: Uses React Router for instantaneous transitions between pages without a full reload.
- **Smart Target**: External links automatically add `target="_blank"` and `rel="noopener noreferrer"` for security and UX.
- **Preview Integration**: Interfaces with the Boltdocs Preview engine via the `boltdocsPreview` prop.

---

## Props (`LinkProps`)

<Field name="to" type="string" required>
  The target URL or path. Can be relative (`/docs/guide`) or absolute (`https://google.com`).
</Field>

<Field name="boltdocsPreview" type="boolean" defaultValue="true">
  Whether to enable the [Link Preview](/docs/core-concepts/link-previews) glassmorphism tooltip when hovering over this link.
</Field>

---

## Personalization (CSS)

Customize the look of your links using these CSS variables in your `custom.css`:

```css
:root {
  --ld-link-color: var(--ld-brand-primary);
  --ld-link-hover-color: var(--ld-brand-secondary);
  --ld-link-decoration: none;
  --ld-link-hover-decoration: underline;
}
```

<Note>
  If `linkPreview` is disabled globally in `boltdocs.config.js`, the `boltdocsPreview` prop will have no effect.
</Note>
