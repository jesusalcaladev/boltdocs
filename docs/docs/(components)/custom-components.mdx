# Custom External Components

## Overview

Because you have Vite and React, you can import **your own** application's components straight into MDX to showcase them. 
This behaves differently from internal components, and thus does not follow the typical component page structure.

<Note>Custom components allow you to use standard React features and bring your specific app UI right into the documentation.</Note>

## Variants & Examples

### Basic Usage

Say you have a `<MyButton>` component defined in `src/components/MyButton.tsx`. You can import it in your MDX like this:

```tsx
import { MyButton } from '../src/components/MyButton';

Here is our primary button:

<MyButton onClick={() => alert('Clicked!')}>Click Me</MyButton>
```

Boltdocs will render the button right alongside your textual documentation. This makes building Live Design Systems incredibly easy.

### Customization

You can combine custom components with the `<ComponentPreview />` component for visual demonstrations. Pass your component as children or to the \`preview\` prop:

```tsx
import { ComponentPreview } from 'boltdocs/client';
import { MyButton } from '../src/components/MyButton';

<ComponentPreview
  preview={<MyButton variant="primary">Try it!</MyButton>}
  code={\`<MyButton variant="primary">Try it!</MyButton>\`}
/>
```

## Best Practices

- **Co-locate with docs**: Keep demo-only components in `docs/src/components/` alongside your documentation.
- **Use aliases**: Configure Vite aliases in `vite.config.ts` to avoid long relative import paths.
- **Export from a barrel**: Create an `index.ts` re-exporting common components for cleaner imports in MDX files.
