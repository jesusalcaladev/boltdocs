# Image

The `<Image />` component allows you to render different images based on the user's active theme (Light or Dark). This is perfect for screenshots or diagrams that have specific color palettes designed for each mode.

## Usage

Use the `theme` prop to indicate when an image should be shown. If no theme is specified, the image will be visible in both modes.

<Image src="https://placehold.co/600x200/1e1e1e/white?text=Dark+Mode+Hero" theme="dark" alt="Architecture" />
<Image src="https://placehold.co/600x200/F4F4F4/dark?text=Light+Mode+Hero" theme="light" alt="Architecture" />

---

## Props (`ImageProps`)

<Field name="src" type="string" required>
  The source path or URL of the image.
</Field>

<Field name="theme" type="'light' | 'dark'">
  If provided, the image will only be rendered when the current site theme matches this value.
</Field>

<Field name="alt" type="string">
  The standard alt text for the image for accessibility****.****
</Field>

---

## Themed Images in **Markdown**

When using standard Markdown syntax `![]()`, Boltdocs resolves to a native `<img>` tag. For theme-specific images, we recommend using the `<Image />` component directly in your MDX files:

```mdx
<Image src="/assets/diagram-dark.png" theme="dark" alt="Architecture" />
<Image src="/assets/diagram-light.png" theme="light" alt="Architecture" />
```
