# Table

The Table component provides a consistent, beautifully styled way to display tabular data. It features glassmorphic headers, subtle row highlighting, and built-in support for sorting and pagination.

## Overview

<Table
  headers={["Feature", "Status", "Version"]}
  data={[
    ["MDX Rendering", <Badge variant="success">Fast</Badge>, "v1.0.0"],
    ["Hot Module Replacement", <Badge variant="info">Enabled</Badge>, "v1.0.0"],
    ["Custom Themes", <Badge variant="warning">Beta</Badge>, "v1.1.0"]
  ]}
/>

## Variants & Examples

### Basic Usage

The simplest way to use a Table is by providing the `headers` and `data` props as arrays.

<Table
  headers={["ID", "User", "Role"]}
  data={[
    ["1", "Alice", "Admin"],
    ["2", "Bob", "Editor"]
  ]}
/>

### Sorting & Pagination

For larger datasets, you can enable interactivity with the `sortable` and `paginated` props.

<Table
  sortable
  paginated
  pageSize={3}
  headers={["Name", "Last Active"]}
  data={[
    ["Charlie", "2 mins ago"],
    ["Diana", "1 hour ago"],
    ["Eve", "Yesterday"],
    ["Frank", "3 days ago"]
  ]}
/>

### Manual Table Structure

If you need full control over the table markup, you can pass standard HTML table elements as children.

<Table>
  <thead>
    <tr>
      <th>Category</th>
      <th>Count</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>Components</td>
      <td>15+</td>
    </tr>
  </tbody>
</Table>

### Personalization (CSS)

Customize the table's appearance globally by overriding these CSS variables in your `custom.css`.

```css
:root {
  /* Header Styles */
  --ld-table-header-bg: var(--ld-bg-soft);
  --ld-table-header-text: var(--ld-text-main);
  
  /* Borders & Rows */
  --ld-table-border: var(--ld-border-subtle);
  --ld-table-row-hover: rgba(255, 255, 255, 0.03);
  --ld-table-stripe-bg: rgba(255, 255, 255, 0.01);
  
  /* Radius */
  --ld-table-radius: 0.5rem;
}
```
