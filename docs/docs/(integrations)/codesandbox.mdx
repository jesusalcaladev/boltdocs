# Interactive Sandboxes

Boltdocs features a high-performance, centralized CodeSandbox API. This allows you to create immersive, live-coding experiences once and reuse them across all your components—perfect for showcasing design systems, complex libraries, or interactive demos.

## 1. Global Configuration

The recommended way to use sandboxes in Boltdocs is by separating your sandbox logic into a dedicated utility file. This keeps your `boltdocs.config.js` clean and allows you to reuse the sandbox configuration in other parts of your application.

### Recommended Pattern

First, create a utility file to define your standard sandbox setup:

```ts
// src/utils/sandbox.ts
import { defineSandbox } from "boltdocs/client";

export const standardSandbox = defineSandbox({
  template: "vite",
  dependencies: {
    "framer-motion": "latest",
    "lucide-react": "latest",
    "boltdocs": "latest"
  },
  files: {
    "src/index.css": {
       content: "@import 'boltdocs/client/index.css';"
    }
  }
});
```

Then, import and use it in your `boltdocs.config.js`:

```js
// boltdocs.config.js
import { standardSandbox } from "./src/utils/sandbox";

export default {
  // ...
  integrations: {
    sandbox: {
      enable: true,
      config: standardSandbox
    }
  }
};
```

<Important>
  When `enable: true` is set in the global configuration, every `CodeBlock` component will automatically display an **Open Sandbox** button that merges the current code into your `standardSandbox`.
</Important>

## 2. Core API

The `boltdocs/client` package provides several functions to interact with CodeSandbox. All functions return an object containing the generated `parameters`, the target `url`, and the original `options`.

### `openSandbox(options: SandboxOptions)`

Launches a sandbox in a new window using a form POST.

```tsx
import { openSandbox } from 'boltdocs/client';

const handleOpen = () => {
  openSandbox({
    title: "Live Demo",
    files: {
      "src/App.tsx": { content: "export default () => <div>Hello World</div>" }
    }
  });
};
```

### `defineSandbox(options: SandboxOptions)`

Generates the parameters and URL without performing any action. Perfect for centralized configurations.

### `embedSandbox(options: SandboxOptions)`

Generates an embeddable iframe URL.

```tsx
import { embedSandbox } from 'boltdocs/client';

export function SandboxEmbed() {
  const { url } = embedSandbox({
    files: { "index.js": { content: "console.log('Embedded Sandbox')" } },
    embed: { view: "preview", theme: "dark" },
  });

  return (
    <iframe 
      src={url}
      style={{ width: '100%', height: '500px', border: '0', borderRadius: '8px', overflow: 'hidden' }}
      title="CodeSandbox Embed"
    />
  );
}
```

## 3. `SandboxOptions` Reference

<Field name="files" type="SandboxFiles">
  A map of file paths to their content.
</Field>

<Field name="dependencies" type="Record<string, string>">
  NPM packages to include in the sandbox `package.json`.
</Field>

<Field name="entry" type="string" defaultValue="'src/App.tsx'">
  The file path where the documentation code snippet will be injected.
</Field>

<Field name="embed" type="SandboxEmbedOptions">
  Configuration for `embedSandbox` view.
</Field>

## 4. CodeBlock Integration

Boltdocs automatically integrates the global sandbox configuration into every code block.

- **Automatic Merging**: Code is injected into the `entry` file of your global config.
- **Per-block Customization**: Override global settings easily.

```mdx
<CodeBlock sandbox={{ title: "Specific Preview" }}>
  {/* Merges with global config but uses this title */}
</CodeBlock>
```

### `SandboxFile`

<Field name="content" type="string | object">
  The content of the file. Objects are automatically stringified.
</Field>

<Field name="isBinary" type="boolean" defaultValue="false">
  Whether the file should be treated as binary.
</Field>

### `SandboxEmbedOptions`

<Field name="view" type="'editor' | 'preview' | 'split'" defaultValue="'split'">
  Which panel to display in the embed.
</Field>

<Field name="theme" type="'dark' | 'light'" defaultValue="'dark'">
  Color theme for the CodeSandbox interface.
</Field>

<Field name="hideNavigation" type="boolean" defaultValue="false">
  Whether to hide the navigation bar in the embed.
</Field>

## CodeBlock Integration

Boltdocs automatically integrates the global sandbox configuration into every code block.

- **Automatic Merging**: When you click the sandbox button in a code block, Boltdocs takes the current code and injects it into the `entry` file (default: `src/App.tsx`) of your centralized configuration.
- **Selective Activation**: If you haven't configured a sandbox globally, the button will not be shown.
- **Manual Overrides**: You can disable or customize the sandbox per-codeblock:

```mdx
<CodeBlock sandbox={false}>
  {/* No sandbox button here */}
</CodeBlock>

<CodeBlock sandbox={{ title: "Specific Component Preview" }}>
  {/* Merges with global config but uses this specific title */}
</CodeBlock>
```

<Tip>
  Use the `entry` property in your global config to specify which file should receive the code from the documentation.
</Tip>
