# Sidebar

The Sidebar is used to organize your documentation into logical groups and pages. It supports nested categories, active state highlighting, and status badges (e.g., "New", "Deprecated").

---

## Architecture

A typical sidebar is composed of a `SidebarRoot` containing several `SidebarGroup` elements, which in turn contain `SidebarLink` elements.

```tsx
import { PrimitiveSidebar } from 'boltdocs/client'
```

---

## API Reference

### PrimitiveSidebar.Root
The main navigation container. It is normally fixed on the left side of the viewport.

<Field name="children" type="ReactNode">
  The groups and links to render.
</Field>

### PrimitiveSidebar.Group
A collapsible or static container for a set of related links.

<Field name="title" type="string">
  The category title (rendered in uppercase).
</Field>

<Field name="icon" type="ElementType">
  Optional Lucide icon to display next to the title.
</Field>

<Field name="isOpen" type="boolean" defaultValue="true">
  Controlled state for collapsing the group.
</Field>

<Field name="onToggle" type="() => void">
  Callback when the group header is clicked.
</Field>

### PrimitiveSidebar.GroupItem
A container for a single link within a group.

<Field name="children" type="ReactNode">
  The link to render.
</Field>

### PrimitiveSidebar.Link
The individual page link.

<Field name="label" type="string" required>
  The page title.
</Field>

<Field name="href" type="string" required>
  The link destination.
</Field>

<Field name="active" type="boolean">
  If `true`, highlighting the link as the current page.
</Field>

<Field name="icon" type="ElementType">
  Optional icon to display before the label.
</Field>

<Field name="badge" type="string | object">
  Status indicator. Supports strings (e.g., `"New"`) or objects (e.g., `{ text: "New", expires: "2025-01-01" }`).
</Field>

## Building from Scratch

Using the `useSidebar` hook, you can iterate over your document groups and render them using the primitives.

```tsx
import { PrimitiveSidebar } from 'boltdocs/client'
import { useSidebar, useRoutes } from 'boltdocs/client'

export function CustomSidebar() {
  const { routes } = useRoutes()
  const { groups, ungrouped } = useSidebar(routes)

  return (
    <PrimitiveSidebar.Root>
      {/* 1. Ungrouped top-level links */}
      {ungrouped.map(route => (
        <PrimitiveSidebar.Link 
          key={route.path}
          label={route.title}
          href={route.path}
          active={route.active}
        />
      ))}

      {/* 2. Grouped links */}
      {groups.map(group => (
        <PrimitiveSidebar.Group key={group.slug} title={group.title}>
          {group.routes.map(route => (
            <PrimitiveSidebar.Link 
              key={route.path}
              label={route.title}
              href={route.path}
              active={route.active}
            />
          ))}
        </PrimitiveSidebar.Group>
      ))}
    </PrimitiveSidebar.Root>
  )
}
```

---

## Logic Hook: `useSidebar`

The `useSidebar` hook transforms the flat array of routes from `useRoutes` into a structured hierarchy based on the metadata defined in your Markdown files.

### Return Value

| Property | Type | Description |
| :--- | :--- | :--- |
| `groups` | `SidebarGroup[]` | Routes organized into named groups. |
| `ungrouped` | `Route[]` | Top-level routes that don't belong to any group. |
| `activeRoute` | `Route \| null` | The currently active route object. |
| `activePath` | `string` | The current normalized URL path. |

## Features

### Badges
Boltdocs supports automatic badge expiration. If you provide an `expires` date, the badge will automatically disappear after that date.

```markdown
---
title: "New Feature"
badge:
  text: "New"
  expires: "2024-12-31"
---
```

### Active State
The active state is calculated automatically based on the URL. If the current URL matches the `href` of a `SidebarLink`, it will receive the `active` styling (primary color background and bold text).

<Note>
  The `icon` prop on `SidebarGroup` and `SidebarLink` expects a **React component** (e.g., `Settings` from lucide-react), not a string. The sidebar converts the icon string to a component internally in the composed `boltdocs/client` Sidebar.
</Note>
