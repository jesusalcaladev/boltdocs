# Navbar

The Navbar in Boltdocs is not a monolithic component but a set of **composable primitives**. This architecture allows you to reorder, replace, or wrap any part of the navigation bar without losing the underlying logic (like active states, localized links, or theme switching).

---

## Architecture

Boltdocs splits the Navbar into several specialized sub-components that you assemble inside a `NavbarRoot`.

```tsx
import {
  NavbarRoot, NavbarContent, NavbarLeft, NavbarRight, NavbarCenter,
  NavbarLogo, NavbarTitle, NavbarLinks, NavbarLink,
  NavbarSearchTrigger, NavbarTheme, NavbarMenu, NavbarItem,
  NavbarSocials, NavbarSplit
} from 'boltdocs/primitives'
```

---

## API Reference

### NavbarRoot
The main `<header>` container. It is `sticky` by default and handles the glassmorphism backdrop.

<Field name="children" type="ReactNode">
  The content to render inside the navbar.
</Field>

<Field name="className" type="string">
  Override default styles. Defaults: `sticky top-0 z-50 w-full border-b backdrop-blur-md`.
</Field>

### NavbarContent
A max-width centered container (`mx-auto max-w-7xl`) that keeps content aligned with the rest of the page layout.

### NavbarLeft, NavbarCenter, NavbarRight
Layout slots using Flexbox to align content.
- **Left**: Typically for Logo and Title.
- **Center**: Typically for Navigation Links (hidden on mobile).
- **Right**: Typically for Search, Theme Toggle, and Socials.

### NavbarLogo
Renders the site logo as a link to the home page.

<Field name="src" type="string" required>
  Path to the logo image.
</Field>

<Field name="alt" type="string">
  Alt text for accessibility.
</Field>

<Field name="width" type="number" defaultValue="24">
  Width in pixels.
</Field>

### NavbarLink
A smart link that knows its active state based on the current URL.

<Field name="label" type="ReactNode" required>
  The link text.
</Field>

<Field name="href" type="string" required>
  The destination path.
</Field>

<Field name="active" type="boolean">
  Manually force the active state. Usually derived from `useNavbar`.
</Field>

<Field name="to" type='"internal" | "external"'>
  Use `"external"` to add an icon and `target="_blank"`.
</Field>

### NavbarSearchTrigger
The button that triggers the search modal. It includes platform-specific keyboard shortcuts (⌘K or Ctrl+K).

<Field name="onPress" type="() => void" required>
  Callback to open your search implementation.
</Field>

### NavbarTheme
A pre-styled theme toggle with smooth icon transitions.

<Field name="theme" type='"dark" | "light"' required>
  Current theme state.
</Field>

<Field name="onThemeChange" type="(isDark: boolean) => void" required>
  Callback to switch themes.
</Field>

## Building from Scratch

If you want to create a bespoke navbar that looks nothing like the default, you can use the `useNavbar` hook to get all the configuration data (links, logo, title) and map it to the primitives.

```tsx
import { 
  NavbarRoot, NavbarContent, NavbarLeft, NavbarRight, NavbarLink 
} from 'boltdocs/primitives'
import { useNavbar, useSearch } from 'boltdocs/hooks'

export function CustomNavbar() {
  const { title, logo, links } = useNavbar()
  const { setIsOpen } = useSearch()

  return (
    <NavbarRoot>
      <NavbarContent>
        <NavbarLeft>
           <img src={logo} alt={title} className="h-8" />
           <span className="font-black italic">{title}</span>
        </NavbarLeft>

        <NavbarRight>
          {links.map(link => (
            <NavbarLink key={link.href} {...link} />
          ))}
          <button onClick={() => setIsOpen(true)}>Search</button>
        </NavbarRight>
      </NavbarContent>
    </NavbarRoot>
  )
}
```

## Logic Hook: `useNavbar`

The `useNavbar` hook is the "brain" behind the navbar. It automatically:
1.  Loads the configuration from `boltdocs.config.ts`.
2.  Handles logo switching between dark/light modes.
3.  Calculates active states for all links.
4.  Translates labels if I18n is enabled.

### Return Value

| Property | Type | Description |
| :--- | :--- | :--- |
| `title` | `string` | The translated site title. |
| `logo` | `string` | The URL for the current theme's logo. |
| `links` | `NavbarLink[]` | Array of processed navigation links. |
| `social` | `SocialLink[]` | Array of social media links. |
| `theme` | `'dark' \| 'light'` | The current UI theme. |

<Tip>
  Always use `useNavbar` when building custom navigation to ensure your site remains configurable via `boltdocs.config.ts`.
</Tip>
