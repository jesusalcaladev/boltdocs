# On This Page (TOC)

The `OnThisPage` primitive renders a table of contents for the current page. It's powered by an `IntersectionObserver` that automatically highlights the heading currently in view and syncs with the scroll position.

---

## Architecture

The table of contents is wrapped in two providers that handle the logic:
- `AnchorProvider`: Manages the list of headings and the active state.
- `ScrollProvider`: (Optional) Automatically scrolls the TOC list when headings are reached.

```tsx
import {
  PrimitiveOnThisPage, AnchorProvider, ScrollProvider
} from 'boltdocs/client'
```

---

## API Reference

### AnchorProvider
The logic provider. Wrap your TOC primitives with this.

<Field name="toc" type="TOCItemType[]" required>
  The array of headings for the current page.
</Field>

<Field name="single" type="boolean" defaultValue="false">
  When `true`, only one heading can be active at a time.
</Field>

### PrimitiveOnThisPage.Root
The main container. Usually sticky on the right side of the page.

### PrimitiveOnThisPage.Header
A styled heading for the TOC section (e.g., "On this page").

### PrimitiveOnThisPage.Content
The scrollable container for the list of headings. Uses a custom scroll mask for a premium feel.

### PrimitiveOnThisPage.List
A `<ul>` container for the heading items.

### PrimitiveOnThisPage.Item
A `<li>` wrapper that handles indentation based on the heading level (h2 vs h3).

<Field name="level" type="number">
  The heading level (2 or 3).
</Field>

### PrimitiveOnThisPage.Link
A smart `<a>` tag that highlights when its corresponding heading is in view.

<Field name="href" type="string" required>
  The anchor id of the heading (e.g., `"#installation"`).
</Field>

<Field name="active" type="boolean">
  Manually force the active state. Usually handled by the `AnchorProvider`.
</Field>

## Building from Scratch

Using the `AnchorProvider` and the `OnThisPage` primitives, you can build a custom TOC that perfectly fits your design.

```tsx
import { AnchorProvider, PrimitiveOnThisPage } from 'boltdocs/client'
import { useRoutes } from 'boltdocs/client'

export function CustomTOC() {
  const { currentRoute } = useRoutes()
  const headings = currentRoute?.headings || []

  return (
    <AnchorProvider toc={headings}>
      <PrimitiveOnThisPage.Root>
        <span className="font-bold mb-4 block">Navigation</span>
        
        <div className="flex flex-col gap-2">
          {headings.map(h => (
            <PrimitiveOnThisPage.Link key={h.url} href={h.url}>
              {h.title}
            </PrimitiveOnThisPage.Link>
          ))}
        </div>
      </PrimitiveOnThisPage.Root>
    </AnchorProvider>
  )
}
```

## Logic Hook: `useActiveAnchor`

If you are building a custom TOC and need to know which heading is currently active (e.g., to animate an indicator), use the `useActiveAnchor` hook.

```tsx
import { useActiveAnchor } from 'boltdocs/client'

function Indicator() {
  const activeId = useActiveAnchor()
  // logic to move a physical indicator to the active link
}
```

### Return Value
`useActiveAnchor` returns the `id` string of the currently active heading (without the `#` prefix).

## Advanced: Scroll Sync

By wrapping `OnThisPageContent` with a `ScrollProvider`, the TOC list will automatically scroll to keep the active item in view. This is useful for very long pages with many headings.

```tsx
import { ScrollProvider, PrimitiveOnThisPage } from 'boltdocs/client'

function TOCBody() {
  const scrollRef = useRef(null)
  
  return (
      <ScrollProvider containerRef={scrollRef}>
         <PrimitiveOnThisPage.Content ref={scrollRef}>
            {/* list of headings */}
         </PrimitiveOnThisPage.Content>
      </ScrollProvider>
  )
}
```

## Full Example

```tsx
import {
  AnchorProvider, ScrollProvider, PrimitiveOnThisPage,
  useActiveAnchor,
} from 'boltdocs/client'
import { useRoutes } from 'boltdocs/client'
import { useRef } from 'react'

function CustomToC() {
  const { currentRoute } = useRoutes()
  const headings = currentRoute?.headings || []
  const scrollRef = useRef<HTMLDivElement>(null)

  // Convert headings to TOC format
  const toc = headings.map((h) => ({
    title: h.text,
    url: `#${h.id}`,
    depth: h.level,
  }))

  return (
    <AnchorProvider toc={toc} single>
      <ScrollProvider containerRef={scrollRef}>
        <PrimitiveOnThisPage.Root>
          <PrimitiveOnThisPage.Header>ON THIS PAGE</PrimitiveOnThisPage.Header>
          <PrimitiveOnThisPage.Content ref={scrollRef}>
            <PrimitiveOnThisPage.List>
              {toc.map((item) => (
                <PrimitiveOnThisPage.Item key={item.url} level={item.depth}>
                  <PrimitiveOnThisPage.Link href={item.url}>
                    {item.title}
                  </PrimitiveOnThisPage.Link>
                </PrimitiveOnThisPage.Item>
              ))}
            </PrimitiveOnThisPage.List>
          </PrimitiveOnThisPage.Content>
        </PrimitiveOnThisPage.Root>
      </ScrollProvider>
    </AnchorProvider>
  )
}
```

<Tip>
  The `AnchorProvider` automatically retries DOM lookups at 100ms, 500ms, and 1000ms after mount. This handles SPA navigations where the MDX content may not be in the DOM immediately.
</Tip>
