# Mejoras para `boltdocs doctor` - Sugerencias de Reparación Automática

## Tarea 1: Sugerencias Inteligentes para Enlaces Rotos
**Prioridad:** Alta  
**Estado:** Pendiente

### Descripción
Implementar búsqueda difusa (Levenshtein) para sugerir el archivo correcto cuando un enlace está roto.

### Checklist
- [ ] Instalar dependencia `fastest-levenshtein`
- [ ] Crear función `findSimilarFile(targetPath, docsDir)` que:
  - [ ] Busque todos los archivos `.md` y `.mdx`
  - [ ] Calcule distancia de edición con nombres de archivo y directorios
  - [ ] Retorne la mejor coincidencia si score &lt; 5
- [ ] Modificar validación de enlaces para usar la sugerencia encontrada
- [ ] Formatear el enlace corregido (relativo vs absoluto según contexto)

### Criterios de Aceptación
- [ ] Si existe un archivo similar, sugerir: `Cambiar a: "/ruta/correcta"`
- [ ] Si no existe similar, sugerir: `Crear archivo en: "ruta/sugerida.md"`
- [ ] Añadir campo `fix` al `Issue` para auto-corrección posterior

## Tarea 2: Sugerencias Contextuales para Frontmatter
**Prioridad:** Media  
**Estado:** Pendiente

### Descripción
Extraer información del contenido para sugerir valores de frontmatter faltantes.

### Checklist
- [ ] Para `title` faltante:
  - [ ] Extraer primer H1 (`/^#\s+(.+)$/m`)
  - [ ] Fallback: nombre del archivo sin extensión
  - [ ] Sugerir: `Añadir al inicio: ---\ntitle: "Valor"\n---`
- [ ] Para `description` faltante:
  - [ ] Extraer primer párrafo no vacío (limpiar frontmatter y headers)
  - [ ] Truncar a 150 caracteres
  - [ ] Sugerir: `Añadir: description: "Primer párrafo..."`

### Criterios de Aceptación
- [ ] Sugerencia incluye el valor exacto a copiar
- [ ] Formato YAML válido en la sugerencia
- [ ] Añadir `fix` con tipo `add_frontmatter`
